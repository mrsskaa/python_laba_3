# Лабораторная работа №3

## Описание проекта

### Данный проект реализует различные алгоритмы и структуры данных на Python:
- Вычисление чисел Фибоначчи (итеративно и рекурсивно)
- Вычисление факториала (итеративно и рекурсивно)
- Алгоритмы сортировки (bubble sort, quick sort, counting sort, radix sort, bucket sort, heap sort)
- Структуры данных: стек и очередь

## Программа запрашивает ввод данных для демонстрации работы всех алгоритмов:
1. Число для вычисления чисел Фибоначчи
2. Массив для сортировки
3. Индексы для quick sort
4. Количество buckets для bucket sort

## Реализованные функции

### Числа Фибоначчи (`src/fibonacci.py`)
- `fibo(n: int) -> int` - вычисление n-го числа Фибоначчи итеративно
- `fibo_recursive(n: int) -> int` - вычисление n-го числа Фибоначчи рекурсивно

### Факториал (`src/factorial.py`)
- `factorial(n: int) -> int` - вычисление факториала итеративно
- `factorial_recursive(n: int) -> int` - вычисление факториала рекурсивно

### Алгоритмы сортировки (`src/sorting.py`)
- `bubble_sort` - сортировка пузырьком
- `quick_sort` - быстрая сортировка
- `counting_sort` - сортировка подсчетом
- `radix_sort` - поразрядная сортировка
- `bucket_sort` - блочная сортировка
- `heap_sort` - пирамидальная сортировка

### Структуры данных

#### Стек (`src/steck_on_list.py`)
Класс `Stack` с методами:
- `push(x)` - добавить элемент в стек
- `pop()` - удалить и вернуть верхний элемент
- `peek()` - посмотреть верхний элемент без удаления
- `is_empty()` - проверить, пуст ли стек
- `__len__()` - получить размер стека

#### Очередь (`src/queue_on_list.py`)
Класс `Queue` с методами:
- `enqueue(x)` - добавить элемент в очередь
- `dequeue()` - удалить и вернуть первый элемент
- `front()` - посмотреть первый элемент без удаления
- `is_empty()` - проверить, пуста ли очередь
- `__len__()` - получить размер очереди

## Обработка ошибок

Проект использует кастомные исключения из `src/errors.py`:
- `DigitisNotNaturalNumberError` - для некорректных входных данных (отрицательные числа, нецелые числа, строки)
- `EmptyError` - для операций с пустыми структурами данных
- `BadTypeError` - для смешанных типов в массивах
- `NotStringError` - для недопустимых типов данных
- `BucketError` - для некорректного количества buckets


## Тестирование

Все функции и классы покрыты unit-тестами с использованием `pytest`. Тесты проверяют:
- Корректность работы в нормальных условиях
- Обработку краевых случаев
- Обработку ошибок и исключений
- Работу с разными типами данных
